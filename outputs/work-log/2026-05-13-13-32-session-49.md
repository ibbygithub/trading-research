## Decisions made

- **Branched off `session-48-exploration-risk-portfolio`, not main.**
  Session 48's chapter set (Parts VI/VII/VIII) was unmerged at
  session start but is a hard prerequisite for the Ch 49 cross-references.
  Follows the precedent set by sessions 41–47.
- **Spec-text-first, code-second order swapped to code-first.** Ch 49
  describes 22 commands; the three new ones needed working code before
  I could write accurate exit-code and output-format sections. Ch 13.4
  remained the ratified contract for `validate-strategy`; Ch 49 only
  fills in operational detail. No deviation from the chapter spec
  resulted.
- **`mode="unknown"` promotion lives in the helper, not the CLI.**
  The session-49 spec asks the CLI to "migrate the 18 pre-session-35
  `mode="unknown"` trials"; the existing helper only set defaults for
  missing fields. I extended `migrate_trials()` itself so any caller
  (CLI, programmatic) gets the same behaviour. Single source of truth.
- **Validate-strategy is a static check for the YAML path only.**
  Registered-template and signal-module dispatch paths report the
  dispatch type but are not imported. Reason: Ch 13.4 spec says the
  command should not execute arbitrary Python; the lint surface stays
  to the declarative ExprEvaluator path. The CLI reports the dispatch
  type so the operator knows the lint was partial.
- **Status pressure thresholds match Ch 56.5 §56.5.6.3 exactly.**
  Green <5 GB, Amber <10 GB, Red <25 GB, Critical otherwise. Tested
  explicitly to lock the contract; if the chapter table is ever
  amended, the test will fail and force the update.
- **§8.6 (feature inventory in `status`) is NOT closed this session.**
  Session 48's work log noted "8.6 deferred to status CLI session,"
  but the session-49 spec scopes the status CLI to "data freshness
  per registered instrument," not the per-feature-set inventory table.
  §8.6 remains [PARTIAL] in the TOC; closing it cleanly needs a
  per-tag freshness table the spec did not call for and is out of
  scope for this session.
- **Synthetic dataset for validate-strategy is uniform numeric
  noise.** It is not a realistic price series; the signal-rate
  warnings (0% / >80%) catch trivially-true and trivially-false
  conditions, not expected trade counts. Documented in Ch 49.15
  under "Limitations" so the operator understands what the warning
  does and does not tell them.
- **Did not touch ruff violations in untouched files.** Pre-existing
  ruff errors number ~417 across `src/`; the session 47 work log
  established that pre-existing violations are not in scope for
  chapter/code sessions. New files ship clean.
