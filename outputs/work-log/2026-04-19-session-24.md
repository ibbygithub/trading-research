## Decisions Made

- **Format conversion:** registry changed from flat JSON array to `{"trials": [...]}` dict. `_load_raw()` handles both, so old readers (`count_trials`, existing tests) are unaffected.
- **Statistical moments as optional fields:** `n_obs`, `skewness`, `kurtosis_pearson` are optional in `Trial` and in `record_trial`. When missing, `compute_dsr` falls back to `n_obs=252, skewness=0, kurtosis_pearson=3` (normal distribution). A warning is logged so callers know the DSR is approximate. This is the right default: better than refusing to compute, worse than using real moments. Session 25 or beyond can backfill these from backtest result objects.
- **DSR result for pre-hardening cohort:** DSR ≈ 5e-135 (essentially 0). This is mathematically correct — all 4 ZN trials had deeply negative Sharpe (best: −9.09). The number is honest.
- **Stationarity CLI:** decided the suite runs as `python -m trading_research.stationarity --symbol 6E`, not as part of the features build. Expensive operation, slow-changing results. This is documented in the design doc as "trigger mechanism."
- **Hurst implementation:** spec chose local R/S implementation over the `hurst` PyPI package (poorly maintained). Dependencies: numpy + scipy.stats.linregress only. Documented in design doc §2.2.
- **Composite classification gate:** all three tests must agree (ADF + Hurst + OU half-life) for `TRADEABLE_MR`. Any individual failure or disagreement routes to a named non-tradeable classification. Cited sources in design doc §4.
