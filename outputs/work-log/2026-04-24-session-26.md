## Decisions Made

1. **Module path `stats/` not `stationarity/`**: session spec took precedence over design doc's module path suggestion. Keeps all statistical primitives in one namespace; avoids a single-purpose sub-package.

2. **Output path `runs/stationarity/`**: design doc's format (parquet + JSON summary + markdown) used; session spec's `outputs/` path was superseded by the more complete design doc spec. Both the parquet (full results DataFrame) and JSON summary (composite dict only) are written.

3. **Hurst R/S behavior clarification**: design doc §8.2 states "AR(1) φ=0.5 should return H < 0.5". This is incorrect for R/S applied to levels: AR(1) with positive φ has positive autocorrelation → H > 0.5 in R/S. H < 0.5 requires negative autocorrelation (oscillating series, AR(1) with φ < 0). Tests were written to reflect the mathematically correct behavior; `test_hurst_ar1_positive_phi_persistent` documents this explicitly. This has trading implications: a VWAP spread with positive autocorrelation at the level (price overshoots before reverting) will NOT show H < 0.45 even if it is ultimately mean-reverting — it will show RANDOM_WALK or INDETERMINATE, which is the honest answer.

4. **OU interpretation field**: set to `MEAN_REVERTING` for the finite half-life case; timeframe-specific labels (TOO_FAST/TOO_SLOW/TRADEABLE) are computed separately in `_interpret_ou` and used in the composite classification.
