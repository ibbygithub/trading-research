## Decisions made

- `safe_prefix()` lives in `multiframe.py` (not `template.py`) because the join caller is responsible for computing the prefix — strategy YAML authors use the output of `safe_prefix()` when writing column names
- Regime filter threshold is fitted on training window in walk-forward mode and auto-fitted on the evaluation set in single-window mode; the data-scientist note in the docstring makes this distinction explicit
- `_build_regime_mask()` uses `isinstance(flt, VolatilityRegimeFilter)` to dispatch to `vectorized_mask()` — this avoids the O(n × Python) overhead of the row-by-row generic fallback while keeping the generic path for future filter types
- `regime_filter:` (singular) and `regime_filters:` (list) are both supported; they are merged before instantiation so one code path handles both
