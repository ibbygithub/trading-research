## Decisions made

### 1. Spec reconciliation — session 35 is plan v3, not the old paper-loop spec
The file `docs/roadmap/session-specs/session-35-paper-loop.md` is from plan v2 and
is superseded by `outputs/planning/sprints-34-38-plan-v3.md`. Session 35 under v3 is
the sweep + leaderboard tooling. This was confirmed by the session 34 work log which
explicitly states "Session 35 contract is unchanged from plan v3."

### 2. Injectable runner design
The `sweep` command's core logic was extracted into `src/trading_research/cli/sweep.py`
with a `runner` parameter that defaults to `_real_runner` but can be replaced by tests.
This avoids real file I/O in tests while fully exercising the sweep orchestration logic.
The existing `backtest` command was NOT refactored to share this runner — scope discipline.

### 3. Backfill semantics
The v3 spec says "backfill existing trials as `mode: 'validation'`." Session 34's two
exploration trials (6A VWAP MR, 6C Donchian BO) have no `mode` field yet, so they
will be backfilled as "validation" by `migrate_trials`. This is correct per spec.
If Ibby wants them tagged as "exploration," re-tag with a one-liner after running
`uv run trading-research sweep` — or manually edit `.trials.json`.

### 4. Leaderboard filter scope
Filters use string equality only (`key=value`). Numeric comparisons (`calmar>1.5`)
are not implemented — the spec only shows equality examples. This is in-scope for
session 37 or later if needed.

### 5. Pre-existing ruff errors in main.py
`ruff check src/ tests/` reports pre-existing issues throughout `main.py` (UP045,
B904, etc.) that existed before this session. New code in the three session-35 files
passes ruff clean. The pre-existing issues are logged here as observed debt.
