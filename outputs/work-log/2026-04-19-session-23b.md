## Decisions made

- `Instrument` type is under `TYPE_CHECKING` guard in `strategies.py` to avoid circular imports at runtime (instruments.py has no dependency on strategies.py; the Protocol methods use `Instrument` only as a type annotation). ruff auto-fixed quoted annotations to bare ones.
- `strategy_class` on `StrategyTemplate` is `type[Any]` not `type[Strategy]` ŌĆö `type[Protocol]` is a known mypy trap that produces false positives. Runtime validity is enforced via `isinstance(instance, Strategy)` inside `instantiate()`.
- Tests each create a fresh `TemplateRegistry()` to avoid contaminating `_GLOBAL_REGISTRY` across test runs.
- `@register_template` decorator populates `_GLOBAL_REGISTRY` on import; tests avoid using it to keep the global clean.
- mypy not installed in this project (not in pyproject.toml) ŌĆö acceptance test satisfied by "if configured" clause in spec.
- `PortfolioContext` is intentionally narrow (open_positions, account_equity, daily_pnl). A comment in the source flags where to extend it in a future session.
