## Decisions made

- **Separate YAML file**: new `configs/instruments_core.yaml` (flat schema) coexists with existing `configs/instruments.yaml` (nested schema) until session 25 consolidates them. This avoids touching the data-layer models which have `extra="forbid"`.

- **Calendar name for ZN**: used `CBOT_Bond` (verified working in existing code) rather than `CMEGlobex_Bond` (spec approximation).
- **`InstrumentRegistry` as class not Pydantic model**: spec intent is a service object, not a data model. Lazy-loaded instance cache, not `@lru_cache`, so different paths can be tested independently.
- **`FeatureSpec` as plain dataclass**: `FeatureSet` is Pydantic with `arbitrary_types_allowed=True` to accept dataclass instances.
- **`code_version` injected at registry load time** via `git rev-parse --short HEAD`; falls back to "unknown" if git unavailable.
- **base-v1.yaml HTF projections**: flattened into per-column `FeatureSpec` entries with `source_tf` key in params. Shift is not in the YAML (implied by code), so not included in hash.
