## 40.0 What this chapter covers

The portfolio report aggregates multiple single-strategy run outputs
into a combined view: headline P&L, per-strategy capital efficiency,
combined equity curve, correlation matrix, sizing comparison, and
drawdown attribution. After reading this chapter you will:

- Know how to invoke the `portfolio` CLI with a list of run IDs
- Understand each section of the HTML report
- Know the data dictionary conventions for portfolio output

This chapter is roughly 3 pages. It is referenced by Chapters 37
(Capital Allocation), 41 (Correlation Analysis), and 42 (Portfolio
Drawdown).

## 40.1 What the portfolio report shows

The HTML portfolio report has eight sections, each generated by a
subsection of
[`eval/portfolio_report.py:18`](../../src/trading_research/eval/portfolio_report.py):

| Section | Content |
|---|---|
| §1 Meta | Strategy list; sizing method used |
| §2 Headline metrics | Combined Calmar, total P&L, max drawdown |
| §3 Capital efficiency | Return on margin per strategy (TradeStation) |
| §4 Combined equity | Plotly equity curve, all strategies + combined |
| §5 Drawdown | Combined drawdown curve; attribution per strategy |
| §6 Correlation | Pearson + Spearman correlation matrix heatmap |
| §7 Sizing comparison | Equal weight vs vol_target vs risk_parity vs inverse_dd |
| §8 Kelly | Kelly fractions for reference (with the "for reference only" warning) |

The report requires at least two valid run IDs. A run ID is the
timestamp subdirectory name under `runs/<strategy_id>/`.

## 40.2 The `eval/portfolio_report.py` module

```python
from pathlib import Path
from trading_research.eval.portfolio_report import generate_portfolio_report

generate_portfolio_report(
    run_ids=["zn-reversion-v1/2026-05-10-14-30", "6a-vwap-fade-v2b/2026-05-10-15-00"],
    output_dir=Path("outputs/portfolio/"),
)
```

The function loads each run's `equity_curve.parquet` and `trades.parquet`
from `runs/<run_id>/`, constructs a `Portfolio` object, and writes
`outputs/portfolio/portfolio_report.html`.

**CLI invocation:**

```
uv run trading-research portfolio \
    --run-id zn-reversion-v1/2026-05-10-14-30 \
    --run-id 6a-vwap-fade-v2b/2026-05-10-15-00 \
    --output outputs/portfolio/
```

The `--run-id` flag accepts full strategy/timestamp paths as they appear
in `runs/`. Partial matching is not supported; the exact directory name
is required.

## 40.3 Reading the portfolio data dictionary

[`eval/data_dictionary_portfolio.py`](../../src/trading_research/eval/data_dictionary_portfolio.py)
documents the output schema. Key fields:

| Field | Type | Meaning |
|---|---|---|
| `strategy_id` | string | Strategy name |
| `return_on_margin` | float | Net P&L / peak margin used (TradeStation) |
| `pearson_corr` | DataFrame | N×N Pearson correlation of daily P&L |
| `spearman_corr` | DataFrame | N×N Spearman correlation of daily P&L |
| `sizing_comparison` | dict | Per-method {calmar, sharpe, final_pnl} |
| `kelly_fractions` | dict | Per-strategy Kelly fractions for reference |
| `drawdown_attribution` | DataFrame | Per-drawdown attribution to strategies |

The portfolio data dictionary is read alongside the Trader's Desk Report
data dictionary from Chapter 17; they are distinct — the per-strategy
report covers a single strategy's backtest, the portfolio report covers
the combined portfolio.

> *Why return on margin rather than return on capital:* for futures
> strategies, capital efficiency is better measured against the margin
> requirement than against a notional capital figure. Two strategies
> with $5,000 net P&L look identical on an equity curve but differ
> dramatically if one required $2,000 of margin and the other required
> $10,000. Return on margin surfaces this.
